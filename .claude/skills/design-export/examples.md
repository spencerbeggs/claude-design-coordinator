## Overview

This document provides complete examples of using the design-export skill
with different formats, options, and scopes.

## Example 1: Export Module to PDF

### Example 1: User Request

> Export effect-type-registry design docs to PDF

### Example 1: Execution Flow

1. Parse parameters: module=`effect-type-registry`, format=`pdf`
2. Load `.claude/design.config.json`
3. Find module design docs path: `.claude/design/effect-type-registry`
4. Glob for markdown files: 3 found
   - observability.md
   - cache-optimization.md
   - architecture.md
5. Read each document with frontmatter
6. Process markdown content
7. Combine documents with master TOC
8. Generate PDF via pandoc + xelatex
9. Validate PDF output
10. Report success

### Example 1: Command Executed

```bash
pandoc combined.md -o effect-type-registry-design-docs.pdf \
  --pdf-engine=xelatex \
  --toc --toc-depth=3 \
  --highlight-style=pygments \
  --variable=geometry:margin=1in \
  --variable=fontsize:11pt \
  --variable=documentclass:article \
  --metadata title="effect-type-registry Design Documentation" \
  --metadata author="Generated by design-export"
```
