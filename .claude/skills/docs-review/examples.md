# Documentation Review Examples

Real-world examples of reviewing user documentation quality.

## Example 1: High-Quality README Review

Reviewing a well-written README that meets all quality standards.

**Input:**

```bash
/docs-review effect-type-registry
```

**Output:**

```text
=== Documentation Quality Report ===

Module: effect-type-registry
Files reviewed: 1 (README.md)

Overall Quality: 92/100 (Excellent) ✅

Readability: 88/100
-----------
Flesch score: 72 (Plain English) ✅
Average sentence length: 16 words ✅
Passive voice: 8% (target: <20%) ✅
Technical jargon: Minimal, well-explained ✅

Completeness: 95/100
------------
Coverage: 95% (19/20 topics) ✅
Missing topics: Advanced error handling (minor)
Required sections: ✅ All present
  - Features ✅
  - Installation ✅
  - Quick Start ✅
  - Usage ✅

Code Examples: 100/100
-------------
Total examples: 4
Valid syntax: 4/4 ✅
Complete (with imports): 4/4 ✅
Copy-paste ready: 4/4 ✅
Comments present: 4/4 ✅

Links: 100/100
-----
Internal links: 6 (all valid) ✅
External links: 3 (all valid) ✅
Anchor links: 2 (all valid) ✅

Recommendations:
- Consider adding advanced error handling section
- Overall excellent documentation quality
```

## Example 5: Strict Mode Review Failing CI

Review in strict mode fails due to warnings.

**Input:**

```bash
/docs-review effect-type-registry --strict
```

**Output:**

```text
=== Documentation Quality Report (Strict Mode) ===

Module: effect-type-registry

Overall Quality: 82/100

⚠️  STRICT MODE: Treating warnings as errors

Warnings found: 3

1. Readability warning:
   - docs/architecture/caching.md: Flesch score 58 (target: >60)

2. Completeness warning:
   - Missing topic: Advanced error handling

3. Link warning:
   - docs/guides/integration.md: External link slow to respond
     (https://example.com - 2.5s)

Exit code: 1 (failure in strict mode)

To pass strict mode:
1. Improve readability of caching.md
2. Add advanced error handling documentation
3. Fix or remove slow external link

Run without --strict to see full report
```
